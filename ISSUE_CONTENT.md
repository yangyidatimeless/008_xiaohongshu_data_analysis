### 3. CI/CD 优化策略

针对允灿提到的 GitHub Actions 时长问题，我会采用以下优化：

| 优化项 | 方案 | 预期效果 |
|--------|------|---------|
| **并行执行** | 后端测试 + 前端测试并行跑 | 总时长从 20 分钟降至 12 分钟 |
| **缓存依赖** | 缓存 node_modules + pip 包 | 每次节省 3-5 分钟 |
| **测试分流** | PR 只跑单元测试，main 分支跑全量 | 减少 70% 的 CI 运行时间 |
| **按需触发** | 文档变更不触发测试 | 减少无效运行 |

**工作流设计**：
```yaml
# PR 流程：只跑单元测试（5 分钟）
on:
  pull_request:
    jobs:
      - backend-unit-test
      - frontend-unit-test

# main/develop 分支：跑全量测试（15 分钟）
on:
  push:
    branches: [main, develop]
    jobs:
      - backend-full-test
      - frontend-full-test
      - security-scan
      - deploy-staging
```

**@允灿**：这样优化后，免费额度（2000 分钟/月）足够用，无需购买。
