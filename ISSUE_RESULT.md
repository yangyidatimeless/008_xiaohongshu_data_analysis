## 🎯 核心结论

### 1. 技术架构决策

| 模块 | 技术选型 | 理由 |
|------|---------|------|
| **后端框架** | FastAPI (Python 3.11) | 异步性能优秀、自动生成 OpenAPI 文档、TypeScript 友好 |
| **前端框架** | Next.js 14 + TypeScript | SSR 支持、类型安全、生态完善 |
| **UI 组件库** | Ant Design 5.x | 组件丰富、中文文档、主题定制 |
| **数据库** | PostgreSQL 15 | 关系型数据、JSON 支持、扩展性强 |
| **缓存** | Redis 7 | 高性能缓存、会话管理、消息队列 |
| **状态管理** | React Query + Zustand | 服务端状态 + 客户端状态分离 |

### 2. API 规范

**统一响应格式**：
```json
{
  "code": 200,
  "data": { ... },
  "message": "success"
}
```

**分页参数**：`page`（页码，从 1 开始）+ `pageSize`（每页数量，默认 20）

**WebSocket 认证**：JWT Token，通过查询参数 `?token=xxx` 传递

### 3. 测试策略

| 测试类型 | 覆盖率目标 | 工具 |
|---------|-----------|------|
| **单元测试** | 80%+ | Pytest (后端) + Vitest (前端) |
| **集成测试** | 50%+ | Pytest + HTTPX |
| **E2E 测试** | 20%+ | Playwright |

**CI/CD 流程**：GitHub Actions 集成单元测试、集成测试、E2E 测试、安全扫描（Bandit + Safety + Trivy）、Lighthouse 性能检测。

### 4. 项目预算
